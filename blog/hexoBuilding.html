发表于 2016-07-27   |   分类于 技术篇

Hexo-GitHub-Gitcafe搭建个人静态博客

Hexo操作命令：

1、新建 hexo new “文件名”
2、编译 hexo generate（也可以简写hexo g）
3、开启本地服务 hexo server（也可以简写hexo s）
4、当修改source文件后，应清空public，重新编译 hexo clean
5、部署到GitHub或Gitcafe（访问速度快）hexo deploy（也可以简写hexo d）

搭建Hexo的过程

因为Hexo是一款基于Node.js的静态博客框架，所以我们还需要安装好它：
下载 Github for windows 和 node.js
打开Git的命令行窗口（先清理npm的缓存，防止安装失败），（npm cache clean）
执行（npm install hexo-cli -g）安装hexo插件
测试是否安装成功（hexo -v），如果出现hexo的版本信息，则说明安装成功
接下来建立一个博客文件目录，并初始化博客目录：
执行（cd e:blog） 和 （hexo init）这两条命令
等命令行安装完成后，有一段提示大致意思是，可以开始使用hexo，则到此安装成功
